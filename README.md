# OpenBD Cコード監視フィード

OpenBD APIを使用してCコードの左から二番目の数字が2の新刊書籍を監視し、RSSフィードを自動生成するシステムです。

## 機能

- Cコード（C + 4桁数字）の左から二番目の数字が2の書籍を監視
- 新規発見書籍をSQLiteデータベースで管理
- RSSフィード自動生成
- GitHub Actionsで1日1回自動実行

## Cコードについて

Cコードは図書分類記号で、4桁の数字で構成されます：
- 1桁目: 対象読者（0=一般、1=教養、2=実用、3=専門、4=婦人、5=学参I、6=学参II、7=児童、8=雑誌、9=コミック）
- 2桁目: 発行形態（0=単行本、1=文庫、2=新書、3=全集・双書、4=ムック・その他、5=事・辞典、6=図鑑、7=絵本、8=磁性媒体など、9=コミック）
- 3-4桁目: 内容分類

このシステムは2桁目が2（新書）の書籍を監視します。

## セットアップ

1. リポジトリをクローン
2. GitHub Actionsが自動実行されるようにします
3. `index.xml`が生成されます

## 手動実行

```bash
python openbd_monitor.py
```

## ファイル構成

- `openbd_monitor.py`: メインスクリプト
- `openbd_books.db`: 書籍データベース（SQLite）
- `index.xml`: 生成されるRSSフィード
- `.github/workflows/monitor.yml`: GitHub Actions設定

## 注意事項

現在のサンプル実装では、固定のISBNリストを使用しています。実際の運用では、新刊ISBNを取得する仕組みが必要です（出版社のデータフィードなど）。
